# 闸门1 效果验收回测规格(Gate-1 Effectiveness Backtest)

> 代码基线:`main @ 0e3ad93`(数据/灯号/防偷看地基已对平,Python official 全周期 `FAIL=0`)。
> **本文件是设计 + 评估规格,不是施工指令。** 第一轮 Codex 只做"只读评估"(见 §7),摸清家底后再分步施工。
> 目的:**第一次真正回答"天气预警到底灵不灵"——而且诚实**(允许结论是"没 edge")。

---

## 0. 定位与心态
- 这是 roadmap 的"闸门1 / alpha 验收",不是又一个 bug 修复。
- **可能的结论之一就是"状态方向性 ≈ 基准、没有可交易 edge"。这不是失败,是诚实**;它会把雷达定位成"环境过滤/风控层"而非"预测器"。
- 因此本回测分**两层**,缺一不可:
  - **A 层(状态本身有没有区分力)**:状态/灯对未来的命中率,诚实对比基准率。
  - **B 层(状态当过滤层有没有用)**:用一个**已知有效的基准策略**(时序动量),比较"一直做" vs "只在雷达亮某灯时做",看夏普/回撤有没有变好。**这才是雷达真正可能有价值的地方。**

## 1. 不可逾越的红线:防偷看(这是整个回测可信的前提)
- **所有用于"在时刻 t 做决策"的统计,只能用 ≤t 的数据。**
- ⚠️ **现有 `router-calibrator` 的 lift/胜率是全样本(in-sample),绝不能直接当 gate1 的结论**——那等于偷看未来。
- 两种合规口径(评估时二选一或都评估):
  - **口径 α(轻、先做)**:只用**因果指标/状态**(`buildIndicatorSnapshots` 的 prefixPercentile/prefixExtrema 已是因果)在 t 的取值 → 看 t+h 的真实结果。测"原始状态有没有预测力",不含校准器。干净、便宜。
  - **口径 β(重、才是真·产品)**:**walk-forward 扩展窗口**——在每个 t 只用 ≤t 重算校准/灯,再看 t+h。测"实际灯号产品"的样本外表现。计算量大(按月/季滚动重算,而非逐根)。
- **参数冻结**:回测期间所有阈值(strongTrendPct、校准分桶、样本闸 30/40…)固定成一个带版本号的 config,中途不许调。

## 2. 状态分类口径(TREND / RANGE / CHAOS)
现有状态名是"放量波动扩张态/震荡网格…",闸门1 需要先映射成三类可验收的环境(**待评估确认映射**):
- **TREND**:gate 主路为 `trendFollowing` 且方向明确(强趋势上行/下行)。
- **RANGE**:gate 主路为 `grid` / 低波动压缩 + 低趋势。
- **CHAOS**:高波动扩张 + 趋势不明 + 余动能矛盾(应被规避)。
- **"命中"定义必须干净、不靠会随波动放大的口径**(网格的 ATR-带子那种定义性成功**不能**直接当命中,需另立判据)。映射与命中判据在评估阶段先定死、文档化。

## 3. 验收指标(对齐之前那张清单)
| 维度 | 指标 | 通过线(草案,可调) |
|---|---|---|
| TREND 命中率 | 趋势灯亮时,趋势方向胜率 | ≥55% **且显著高于基准率** |
| RANGE 命中率 | 网格灯亮时,区间维持率(干净判据) | ≥55% 且高于基准 |
| CHAOS 规避 | 在 CHAOS 灯期间交易 vs 全程 | CHAOS 期表现**明显更差**(证明值得规避) |
| 跨品种稳定 | 上述按 `periodWeight` 加权、不靠 2–3 个币 | 多数厚历史品种一致 |
| 跨年稳定 | 按年份/regime 分段(2019/2021牛/2022熊/2024…) | 不只在某一年有效 |
| 切换滞后 | 状态翻转 vs 行情实际拐点差几根 K | 不慢到信号已走完 |
| 过滤层增益(B 层) | 雷达过滤的动量 vs 纯动量 | 夏普↑ / 最大回撤↓ |
| 统计可信度 | **Deflated Sharpe / PBO**(López de Prado) | DSR>0、PBO 低 |
| 无偷看 | 全程 walk-forward / 因果 | 必须满足,否则结果作废 |

## 4. 可复用 vs 需新写(初判,待评估核实)
**已有、可复用**:
- `strategy-router-backtest.mjs` 的 `observationRows`:每个 (state/route, horizon) 已带 `futureReturnPct / success / directionalWin / routeReturnPct / scoreBucket`——A 层命中率的原料。
- `periodWeight`(H5)——跨品种加权现成。
- 因果指标(prefixPercentile 等)——口径 α 直接可用。
**需新写**:
- TREND/RANGE/CHAOS 映射 + 干净命中判据。
- **按年份/regime 分段**汇总。
- **净值曲线 + 最大回撤 + Deflated Sharpe / PBO** 计算(目前只有胜率/平均收益,没有净值与过拟合校正)。
- **B 层**:时序动量基准策略(Moskowitz 式,12 月/可配)+ "雷达过滤 vs 不过滤"对照。
- **walk-forward 框架**(口径 β):扩展窗口滚动重算。

## 5. 输出
- `reports/gate1_*`:按品种/周期/年份的命中率与净值;过滤层对照;DSR/PBO。
- 一份**结论页**:三类命中率、过滤层增益、是否过闸门1,**诚实写明"作为预测器/作为过滤层"各自的结论**。

## 6. 边界
- **不碰**已对平的 Python official 生产报告与研究算法;gate1 是**独立的离线评估层**(读历史 + 自己算),失败/重跑不影响生产。
- 优先 Python 实现(与现产线同语言);可复用 `backend_py/research` 的因果指标。

## 8. 心理预期(写给用户)
- 闸门1 很可能显示 **A 层方向命中接近基准**(M2 已暗示 ~45–46%)。**这是诚实结果**。
- 真正的价值大概率在 **B 层(过滤层增益)**:即便方向不准,"避开 CHAOS / 只在合适环境上场"仍可能改善基准策略的夏普/回撤。
- **闸门1 的目的不是"证明它行",而是"诚实知道它行不行、以及以什么方式行"。** 跑出"不行"也是巨大价值——省得你在没 edge 的东西上加杠杆。
