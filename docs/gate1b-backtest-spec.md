# 闸门1-B 效果验收:过滤层增益(Gate-1B Filter-Value Backtest)

> 代码基线:`main @ 5b6d5c3`。承接 G1-A 结论:**雷达不预测方向(TREND lift≈0/负),但能识别波动环境(RANGE lift +5~+13;CHAOS=当下描述、非未来预测)。**
> 本文件是设计 + 拍板规格。第一轮 Codex 只做"只读细化"(§7),拍板后再写 G1-B 代码。
> **目的:验证"用雷达挑环境上场"能不能把一个基准策略的风险调整后收益变好——不押方向。**

---

## 0. 核心问题(就一句)
> **"把一个简单、已知的基准策略,只在雷达说'合适'的环境里跑" vs "不挑环境一直跑",夏普↑ / 最大回撤↓ 吗?**
> 这才是把 G1-A 的"识别力"变成"有用"的一步。**赢的是过滤,不是预测。**

## 1. 测哪几组(对应 G1-A 的两个发现)
**S1 — RANGE 收割(利用 RANGE 的正 edge)**
- 基准:一个**简单因果均值回归**(如:价偏离中值 N 个 ATR 时反向、到中值或 h 根后平),**不挑环境一直做**。
- 过滤版:**只在 RANGE 灯亮时**开新仓。
- 预期:过滤版夏普/回撤更好(因为 RANGE 有 +edge)。

**S2 — 躁动规避(利用 CHAOS=当下高波动的描述性)**
- 基准:一个**简单因果时序动量**(Moskowitz 式:按过去 k 期收益符号持仓),不挑环境。
- 过滤版:**CHAOS 灯亮时空仓/减仓**(降杠杆),其余照做。
- 预期:回撤更小(避开当下高波动段);夏普未必升,但**尾部风险**应改善。

**S3(诚实对照)— TREND 过滤(预期会失败,但要测)**
- 同 S2 动量,过滤版"只在 TREND 灯亮时做"。
- 预期:**可能无改善甚至变差**(G1-A 显示 TREND 方向无 edge)。**测它正是为了证明"别用方向那条"——一个干净的负结果同样有价值。**

**对照基准**:每个品种的 **买入持有(Buy&Hold)** 和 **不过滤版**,三者并列比。

## 2. 防偷看红线(与 G1-A 同等严格)
- **t 时刻的开/平仓决策只用 ≤t 数据**:雷达状态用 G1-A 的因果 α 口径;入场用 `close[t]`,出场用规则(到中值/反向信号/持有 h 根)。
- **仓位/波动缩放只能用滞后波动**(如 ATR[t] 或 ≤t 的已实现波动),不用未来。
- **参数冻结、不许在测试期调参**:所有阈值(偏离 N、持有 h、动量 k、缩放目标)**事前定死**,写进带版本号的 config;若要选参,只能在**训练段**选、验证段不动(本轮建议直接预设、不调,避免过拟合)。
- **成本要进**:手续费 + 滑点作为参数(默认给个保守值,如单边 5–10bps),净值用**含成本**口径。
- 禁读任何 calibration 产物 / 未来函数。

## 3. 指标(过滤版 vs 不过滤 vs 买入持有)
每个 (品种, 周期, 策略) 三条净值并列,报:
| 指标 | 说明 |
|---|---|
| CAGR / 总收益 | 含成本 |
| **Sharpe / Sortino** | 主看 |
| **最大回撤 MaxDD** | 主看 |
| 暴露率 exposure | 过滤版在场时间占比(过滤会降暴露,要看"少做但更优") |
| 换手 turnover | 成本敏感性 |
| 胜率 / 盈亏比 | 辅助 |
| **过滤增益** | 过滤版 −(不过滤版):ΔSharpe、ΔMaxDD |
- **跨品种汇总按 `periodWeight` 加权**;`observations/交易数 < N` 的标 `lowSample`、不进主结论。
- **组合层**:把过滤版各品种等风险/等权拼成一个组合净值,看分散后的夏普/回撤(单品种噪声大,组合才看得出真信号)。

## 4. 防过拟合(轻量,重的留 G1-C)
- 本轮**不做参数搜索**(预设参数),从源头压过拟合。
- 报 **Deflated-Sharpe 的意识**:若 S1 过滤增益看着好,标注"未做多重检验校正,正式结论待 G1-C 的 DSR/PBO"。
- **跨年分段**:过滤增益按年份/regime 看,别只在某一年成立。

## 5. 输出
- `reports/gate1b_equity_<strategy>.csv`(各品种三条净值)、`reports/gate1b_summary.csv`(过滤 vs 不过滤 vs B&H 的 Sharpe/MaxDD/增益)、`reports/gate1b_portfolio.csv`(组合层)。
- 大中间产物(逐笔/逐根)**加 .gitignore,别提交**(教训:G1-A 的 observations.csv 866MB)。
- 结论页:**S1/S2/S3 各自"过滤有没有让基准变好",诚实写明赢在哪、哪条没用。**

## 6. 边界
- **独立离线层**:只读 `data/clean`,只新增 `gate1b_*.py` + `reports/gate1b_*`;**不碰生产报告、不动研究算法、不读 calibration**。
- 复用 G1-A 的因果状态 + 现成 `d8/d13/d21/d34/trendScore` 做动量,`middleDeviationAtr` 做均值回归。

## 8. 心理预期(写给用户)
- **S1(RANGE 收割)最可能赢**——G1-A 已证 RANGE 有 +edge,过滤大概率改善夏普/回撤。
- **S2(避 CHAOS)可能降回撤**——避开当下高波动段,尾部更稳。
- **S3(TREND 过滤)很可能没用甚至变差**——这正好坐实"别用方向那条",**负结果也是结论**。
- **赢的标准是"过滤版比不过滤版更优",不是"绝对赚钱"**;能不能上真钱还要看成本、容量、G1-C 的过拟合校正。**这一跑,第一次回答"识别环境能不能转化成实打实的好处"。**
