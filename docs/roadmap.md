## 当前状态(2026-06 封板)

> **决策:封板,把雷达当监控/风控背景层用,停止追 alpha。** 详见 `docs/gate1-conclusion.md`。

- **闸门 0(基座可信)✅ 过了**:Python↔Node 全周期逐字对平 `FAIL=0`、防偷看(因果指标/walk-forward)、冻结 golden 回归(1D/4H/8H/1W,`PASS=48 FAIL=0`)、`verification-log.md` 已用真实数字填满。
- **闸门 1(单信号 OOS 有 edge)❌ 诚实没过**:Gate-1 实测——
  - **A 层(区分力)**:方向命中 ≈ 基准率(雷达不预测涨跌);但能识别波动环境(RANGE/CHAOS 描述靠谱)。
  - **B 层(过滤增益)**:S1 RANGE 收割的"赢"是暴露塌缩假象、S2 避 CHAOS 中性、S3 TREND 过滤确认有害。**无任何已证实的可交易 edge。**
- **结论**:雷达 = **靠谱的环境/波动描述器 + 风控背景层**,不是 alpha 源、不是预测器。**不在没 edge 的东西上加杠杆——这是 Gate-1 最大的价值。**
- **未继续往 2→8 盖楼**:roadmap 第 78 行"真正缺的下一块是 alpha"已被 Gate-1 诚实回答(现在没有)。在无 edge 的基座上加杠杆 = 文献反复警告的死法,故主动收手。

## 关键认知(华尔街经验)

1. **雷达不是"交易系统",是"条件层"。** 它横跨 **3(元标注)、5(组合)、6(风控)**——不产生交易,而是决定"现在该不该让某类策略上场、给多少风险"。这是 AQR / CTA / Bridgewater 用 regime 的标准方式。把它直接当买卖信号 = 之前文献反复警告的死法。

2. **真正缺的下一块是第 2 步:alpha。** 雷达说"现在适合趋势",但"趋势里具体怎么进出"才是赚钱的 edge,目前还没有。没有 edge,光有环境过滤等于零。

3. **闸门 1 是生死线。** López de Prado 那套(purged CV、Deflated Sharpe、PBO)就是为这关造的——自以为有效的策略,过了这关多半现原形。

4. **regime 只能描述当下、不能预测切换**(Dacco & Satchell 1999)。所以雷达永远是"过滤/加权",不是"预言"。

## 最诚实的"下一小步" —— ✅ 已做完(负结果),据此封板

原计划:**用雷达去"调味"一个已知的简单 edge**,验证它到底加不加分:

> 拿学术公认稳的信号 **时序动量(Moskowitz 2012)** 做基线,问一个问题:
> **"只在雷达亮趋势绿灯时做动量,跟一直做动量比,夏普 / 回撤有没有变好?"**

**这一步已经做了**(Gate-1A/1B,见 `docs/gate1-conclusion.md` 与 `gate1b-backtest-spec.md`)。答案:

- TREND 过滤(S3)**变差**(ΔSharpe 全负);CHAOS 规避(S2)**中性**;RANGE 收割(S1)的正增益是**暴露塌缩假象**(过滤后几乎不交易)。
- 即"只在雷达亮灯时做动量"**没有跑赢一直做**——**雷达的方向 edge 不存在**。

**这正是"可证伪的下一步"该有的样子:它诚实地证伪了"雷达能直接转化成钱"。** 一个干净的负结果,省得在没 edge 的东西上加杠杆。

**因此项目封板**:闸门 0 已过、闸门 1 诚实没过 → 雷达定位为**监控/风控背景层**,不再往 2→8 盖楼。若将来要重启 alpha,唯一值得给一次干净机会的是"用与 RANGE 不矛盾的载体(区间内网格/边界回归)重测 RANGE 收割",并补 Deflated-Sharpe/PBO——但那是另一次明确的立项,不是本项目的尾巴。
