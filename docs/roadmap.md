## 关键认知(华尔街经验)

1. **雷达不是"交易系统",是"条件层"。** 它横跨 **3(元标注)、5(组合)、6(风控)**——不产生交易,而是决定"现在该不该让某类策略上场、给多少风险"。这是 AQR / CTA / Bridgewater 用 regime 的标准方式。把它直接当买卖信号 = 之前文献反复警告的死法。

2. **真正缺的下一块是第 2 步:alpha。** 雷达说"现在适合趋势",但"趋势里具体怎么进出"才是赚钱的 edge,目前还没有。没有 edge,光有环境过滤等于零。

3. **闸门 1 是生死线。** López de Prado 那套(purged CV、Deflated Sharpe、PBO)就是为这关造的——自以为有效的策略,过了这关多半现原形。

4. **regime 只能描述当下、不能预测切换**(Dacco & Satchell 1999)。所以雷达永远是"过滤/加权",不是"预言"。

## 最诚实的"下一小步"(等基础完善后再启动)

不要急着搭 2→8 全套。**先用雷达去"调味"一个已知的简单 edge**,验证它到底加不加分:

> 拿学术公认稳的信号 **时序动量(Moskowitz 2012)** 做基线,问一个问题:
> **"只在雷达亮趋势绿灯时做动量,跟一直做动量比,夏普 / 回撤有没有变好?"**

这一步小、可证伪、直接回答"我的雷达到底有没有用",做完顺带过了闸门 0/1。它是从雷达迈向交易系统的第一块踏脚石。

**前置条件(必须先完成):** M3 / L5 收尾 + `verification-log.md` 用真实本地数字填满(闸门 0)。在基座未验证可信前,不要往上盖楼。
